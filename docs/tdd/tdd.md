## Overview

The `tdd/` directory holds Technical Design Documents (TDDs). A TDD describes *how* a feature will be built — proposed approach, module design, data flow, key trade-offs, and alternatives considered. TDDs are created during Stage 2b of the [[workflow]] and updated in Stage 3b when implementation diverges from the design.

See the [[workflow]] for when to write a TDD vs. proceeding directly to implementation.
